# Генератор договоров из docx-шаблонов

Заполняете форму → получаете готовый пакет: договор, приложения (каждое — отдельным
файлом), акт, согласие на ПД, счет. Реквизиты контрагентов сохраняются в базу
и в следующий раз подставляются из выпадающего списка.

## Два способа запуска

### ☁️ Облачный — основной. Локально НИЧЕГО не устанавливается

Код лежит в GitHub-репозитории, форма работает на бесплатном сервере Streamlit,
а все «живые» данные — на вашем Яндекс Диске. Открываете ссылку с любого
устройства (хоть с телефона), вводите пароль и работаете.

Как это устроено после настройки:

1. Открываете `https://ваше-имя.streamlit.app` → пароль.
2. Форма сама подтягивает с Диска свежие шаблоны и базу контрагентов.
3. Заполняете поля → «Сгенерировать».
4. На Диске появляется папка `Документы/№X — Исполнитель/` с готовыми
   файлами, форма показывает ссылку «поделиться». База и журнал
   обновляются там же.

GitHub нужен только как место хранения кода — заглядывать туда после
настройки не придется. Шаблоны правятся прямо на Диске (Word или Яндекс
Документы), репозиторий при этом не трогается.

Настройка за ~10 минут — раздел «Облачный режим» ниже.

### 💻 Локальный — запасной, по желанию

Если хочется погонять на своем компьютере без облака: установите Python 3.10+
(python.org, отметьте «Add to PATH»), затем в папке проекта:

```
pip install -r requirements.txt
streamlit run app.py
```

Откроется страница в браузере; без токена Диска пакет скачивается zip-архивом,
с токеном (его можно ввести в боковой панели) — синхронизируется с Диском так
же, как облачный вариант.

Без браузера тоже работает: см. `test_run.py` — пример вызова
`core.generate_package()` из кода (можно гонять пачками из Excel и т.п.).

## Структура

```
app.py            — форма (Streamlit)
core.py           — движок: рендер шаблонов, суммы/даты прописью, база контрагентов
templates/        — docx-шаблоны (правятся обычным Word'ом)
  dogovor.docx        договор
  pril1_perechen.docx Приложение 1 — Перечень услуг
  pril2_zadanie.docx  Приложение 2 — Задание
  akt.docx            Приложение 3 — Акт
  soglasie_pd.docx    Приложение 4 — Согласие на обработку ПД
  schet.docx          Счет на оплату
  tochka_prilozhenie.docx  «Точка»: приложение к договору
  tochka_akt.docx          «Точка»: акт сдачи-приёмки выполненных работ
  tochka_schet_oferta.docx «Точка»: счёт-оферта (100% аванс)
ydisk.py          — клиент REST API Яндекс Диска (облачный режим)
Dockerfile        — для развертывания на сервере
контрагенты.xlsx  — база: лист «Контрагенты» + лист «Журнал».
                    АО «Точка» уже сохранено — выберите его в выпадающем
                    списке формы, и реквизиты подставятся сами.
build_templates.py — скрипт, которым собраны шаблоны (нужен только для пересборки)
```

В форме у Заказчика и Исполнителя есть выпадающий список «Подставить
сохранённого контрагента»: выбор сразу заполняет все поля. Новые контрагенты
попадают в список автоматически после генерации (галочка «Сохранить…»).

## Как править шаблоны

Открываете docx в Word и редактируете как обычный текст. Разметка:

| Разметка | Что делает |
|---|---|
| `{{ переменная }}` | подставляется значение |
| `{%p if блок_нда %}` … `{%p endif %}` | условный блок: каждый тег — **отдельным абзацем**, эти абзацы-маркеры удаляются из результата |
| `{%tr for у in услуги %}` … `{%tr endfor %}` | цикл по строкам таблицы: теги — в **отдельных строках таблицы** до и после строки-образца |

Важно: тег нельзя «разрывать» — если Word подчеркнул его проверкой правописания и
что-то сломалось, сотрите тег целиком и наберите заново одним куском.

### Основные переменные

Стороны: `заказчик_наименование`, `заказчик_должность_род`, `заказчик_подписант_род`,
`заказчик_основание_род`, `заказчик_подписант_кратко`, `заказчик_инн`, `заказчик_огрн`,
`заказчик_адрес`, `заказчик_реквизиты` (многострочный блок); аналогично
`исполнитель_фио`, `исполнитель_фио_кратко`, `исполнитель_инн`, `исполнитель_паспорт`,
`исполнитель_адрес`, `исполнитель_реквизиты`.

Договор: `договор_номер`, `договор_дата`, `город`, `услуги_начало`, `услуги_окончание`,
`стоимость_цифрами`, `стоимость_прописью`, `предоплата_процент`, `постоплата_процент`,
`штраф_чек_процент`, `штраф_уведомление`, `акт_номер`, `акт_дата`, `счет_номер`,
`счет_дата`, `согласие_до`.

Сроки в днях (и пропись в родительном падеже — добавьте `_проп`): `справка_дней`,
`пояснения_дней`, `акт_дней`, `приемка_дней`, `устранение_дней`, `предоплата_дней`,
`постоплата_дней`, `расторжение_дней`, `претензия_дней`, `фм_уведомление_дней`.
Пример: `{{ акт_дней }} ({{ акт_дней_проп }}) рабочих дней` → «5 (пяти) рабочих дней».

Блоки и номера разделов: `блок_нда`, `блок_фм`, `блок_согласие`; так как блоки меняют
нумерацию, заголовки после раздела 6 используют переменные `с_нда`, `с_фм`, `с_раст`,
`с_спор`, `с_проч`, `с_прил`, `с_рекв` — номера считает программа.

Цикл услуг: внутри `{%tr for у in услуги %}` доступны `{{ у.номер }}`,
`{{ у.наименование }}`, `{{ у.колво }}`, `{{ у.цена }}`, `{{ у.стоимость }}`
(= кол-во × цена), `{{ у.начало }}`, `{{ у.окончание }}`, `{{ у.требования }}`.

Документы «Точка»: `прил_номер`, `прил_дата`, `оферта_срок_оплаты`,
`оферта_срок_работ`, `оферта_результат`, `оферта_формат`, `ндс_строка`,
`ндс_предложение`, `исполнитель_тип` («Самозанятый»/«ИП»), `исполнитель_огрн`,
`заказчик_кпп`, блок `блок_ис` (пункт об интеллектуальной собственности —
снимите галочку, если объектов ИС не возникает).

⚠️ НДС: для самозанятого строка НДС подставляется автоматически («не облагается,
НПД»). Для ИП по умолчанию пишется «без НДС» — с 2025 года применение УСН само
по себе не освобождает от НДС, поэтому при необходимости укажите ставку и сумму
в поле «Строка НДС» формы.

## Чтобы добавить свой блок (например, «Гарантии»)

1. В Word вставьте в `dogovor.docx` абзац `{%p if блок_гарантии %}`, текст блока,
   абзац `{%p endif %}`.
2. В `core.py` в `build_context` добавьте `"блок_гарантии": bool(d.get("блок_гарантии"))`
   (и при необходимости — номер раздела в расчет `sec`).
3. В `app.py` добавьте галочку и передайте значение в `data`.

## Облачный режим: сервер + Яндекс Диск

Форма крутится на сервере (работать можно хоть с телефона), а всё «состояние»
живет на вашем Яндекс Диске:

```
Диск/Генератор договоров/
  Шаблоны/            ← docx-шаблоны; правьте их прямо на Диске (Word или
                        Яндекс Документы) — генератор берет свежие версии
  Документы/          ← готовые пакеты, по папке на договор, со ссылкой
                        «поделиться»
  контрагенты.xlsx    ← общая база контрагентов и журнал
```

При первом подключении приложение само создаст папки и загрузит на Диск
стартовые шаблоны (ваши правки на Диске никогда не перезаписываются).

### Шаг 1. Получить OAuth-токен Диска (5 минут)

1. Откройте oauth.yandex.ru → «Создать приложение» → платформа «Веб-сервисы»,
   Redirect URI: `https://oauth.yandex.ru/verification_code`.
2. В доступах отметьте «Яндекс.Диск REST API»: чтение и запись.
3. Скопируйте ClientID и откройте в браузере:
   `https://oauth.yandex.ru/authorize?response_type=token&client_id=<ClientID>`
   — разрешите доступ, токен покажется на странице. Храните его как пароль.

### Шаг 2. Развернуть форму

**Вариант А — Streamlit Community Cloud (бесплатно, без своего сервера).**
Выложите папку проекта в GitHub-репозиторий (можно приватный) → зайдите на
share.streamlit.io → New app → выберите репозиторий и `app.py` → в Settings →
Secrets вставьте:

```
YANDEX_DISK_TOKEN = "ваш токен"
APP_PASSWORD = "пароль для входа в форму"
```

Получите постоянную ссылку вида `https://…streamlit.app`. После простоя
приложение «засыпает» и просыпается при открытии за ~30 секунд — для
внутреннего инструмента это нормально.

**Вариант Б — свой сервер / Yandex Cloud (Docker).**

```
docker build -t dogovory .
docker run -d -p 8501:8501 \
  -e YANDEX_DISK_TOKEN="ваш токен" \
  -e APP_PASSWORD="пароль" dogovory
```

### Безопасность

- `APP_PASSWORD` обязателен для публичного адреса: в форме персональные данные.
- Токен Диска = полный доступ к Диску, не коммитьте его в git
  (`.gitignore` уже настроен) и не пересылайте в мессенджерах.
- База контрагентов «последняя запись побеждает»: если два человека нажмут
  «Сгенерировать» одновременно, сохранится версия второго. Для 1–3
  пользователей это не проблема.

Напомню: локальный запуск — только опция для желающих, для облачного режима
он не нужен.
