# Генератор договоров из docx-шаблонов

Выбираете комплект (Самозанятые / ИП / ЮЛ) → заполняете форму → получаете пакет
документов на Яндекс Диске со ссылкой «поделиться». Контрагенты хранятся в общей
базе с историей изменений.

## Два способа запуска

### ☁️ Облачный — основной. Локально НИЧЕГО не устанавливается

Код в GitHub, форма на сервере Streamlit, все данные — на Яндекс Диске.
Работает с любого устройства у всей команды (5–7 человек — нормально).

После настройки:
1. Открываете `https://ваше-имя.streamlit.app` → пароль → в боковой панели
   вводите своё имя (попадает в журнал и историю изменений).
2. Выбираете комплект шаблонов, заполняете форму, «Сгенерировать».
3. На Диске появляется `Документы/СЗ-15 — Иванов И.И./` с файлами, форма
   показывает ссылку «поделиться» и очищается для следующего пакета.

Настройка за ~10 минут — раздел «Облачный режим» ниже.

### 💻 Локальный — запасной

Python 3.10+, затем: `pip install -r requirements.txt` и `streamlit run app.py`.
Без токена Диска используются встроенные шаблоны, пакет скачивается zip-архивом.

## Структура на Яндекс Диске

```
Диск/Генератор договоров/
  Шаблоны/
    Самозанятые/   Акт сдачи-приёмки.docx, Приложение к договору.docx,
                   Счёт-оферта.docx, манифест.json
    ИП/            то же, формулировки для ИП
    ЮЛ/            то же, с НДС (ставка, сумма, итого с НДС)
  Документы/       готовые пакеты, по папке на номер
  контрагенты.xlsx листы «Контрагенты», «Архив» (история версий), «Журнал»
```

**Комплекты — это просто папки.** Создали на Диске папку «Шаблоны/Подряд»,
положили docx — в форме появился новый комплект с галочками по числу файлов.
Каждый docx в папке = отдельная галочка «какие документы сгенерировать».

`манифест.json` (необязателен, без него значения угадываются по имени папки):

```json
{ "префикс": "СЗ", "тип_исполнителя": "Самозанятый", "ндс": false }
```

- `префикс` — нумерация документов комплекта: СЗ-1, СЗ-2… (счетчик ведется по
  журналу; поле «Номер» в форме можно править руками);
- `тип_исполнителя` — Самозанятый / ИП / ЮЛ: переключает набор полей
  исполнителя (паспорт ↔ ОГРНИП ↔ КПП и директор);
- `ндс: true` — в форме появляется ставка НДС, в контексте — суммы с НДС.

Кнопка «🧪 Проверить комплект» прогоняет все шаблоны выбранной папки на
тестовых данных и показывает, в каком файле какой тег сломан — полезно после
правок шаблонов.

Старый стартовый комплект (договор с самозанятым + 4 приложения) лежит в
`templates/_архив_старый_комплект/` и в форме не показывается; чтобы вернуть —
переименуйте папку без подчеркивания и положите на Диск.

## База контрагентов: версии и защита от затирания

- Ключ записи — **ИНН + статус**. Реквизиты (банк, адрес, ОГРН, директор)
  можно менять; смена статуса (самозанятый → ИП) дает параллельную запись.
- Если при генерации данные в форме отличаются от записи в базе, форма
  показывает диалог с разницей по полям: **«Обновить запись (старая версия —
  в Архив)»** или **«Не сохранять — только в этот документ»**.
- Лист «Архив» хранит все прежние версии: когда заменено и кем. Внизу формы —
  «📜 История изменений»: выбираете контрагента, версию, «Вернуть» — текущая
  при этом тоже уходит в архив, потерять данные нельзя.
- ИНН/ОГРН/БИК проверяются по контрольным цифрам; расхождения показываются
  предупреждением (не блокируют генерацию).
- Перед каждой записью база перечитывается с Диска — одновременная работа
  команды безопасна (окно конфликта — секунды).

## Разметка шаблонов

Открываете docx (в Word или прямо на Диске) и правите как текст:

| Разметка | Что делает |
|---|---|
| `{{ переменная }}` | подставляется значение |
| `{%p if блок_ис %}` … `{%p endif %}` | условный блок: каждый тег — отдельным абзацем |
| `{%tr for у in услуги %}` … `{%tr endfor %}` | цикл по строкам таблицы: теги в отдельных строках до/после строки-образца |

Тег нельзя разрывать: если сломалось после правки — сотрите тег и наберите
заново целиком.

### Основные переменные

Документ: `договор_номер`, `договор_дата`, `город`, `услуги_начало`,
`услуги_окончание`, `акт_номер`, `акт_дата`, `счет_номер`, `счет_дата`,
`прил_номер`, `прил_дата`.

Заказчик: `заказчик_наименование`, `_инн`, `_кпп`, `_огрн`, `_адрес`,
`_подписант_род`, `_основание_род`, `_подписант_кратко`, `_реквизиты`
(многострочный блок). Исполнитель: те же с префиксом `исполнитель_`, плюс
`исполнитель_фио`, `_фио_кратко`, `_паспорт`, `_банк`, `_бик`, `_счет`,
`_корсчет`, `_должность_род` (для ЮЛ).

Деньги: `стоимость_цифрами`, `стоимость_прописью`, `ндс_ставка`, `ндс_сумма`,
`итого_с_ндс_цифрами`, `итого_с_ндс_прописью`, `ндс_строка`, `ндс_предложение`.

Цикл услуг: `{{ у.номер }}`, `{{ у.наименование }}`, `{{ у.колво }}`,
`{{ у.цена }}`, `{{ у.стоимость }}` (= кол-во × цена), `{{ у.начало }}`,
`{{ у.окончание }}`.

Поля счёта-оферты: `оферта_срок_оплаты`, `оферта_срок_работ`,
`оферта_результат`, `оферта_формат`. Блок: `блок_ис`.

⚠️ НДС: для самозанятого строка НДС подставляется автоматически («без НДС,
НПД»), для ИП — «без НДС», для ЮЛ — по ставке из формы. С 2025 года УСН сама
по себе не освобождает от НДС — при необходимости правьте поле «Строка НДС».

## Облачный режим: настройка

### Шаг 1. OAuth-токен Диска (5 минут)

1. oauth.yandex.ru → «Создать приложение» → платформа «Веб-сервисы»,
   Redirect URI: `https://oauth.yandex.ru/verification_code`.
2. Доступы: «Яндекс.Диск REST API» — чтение и запись.
3. Откройте `https://oauth.yandex.ru/authorize?response_type=token&client_id=<ClientID>`
   → «Разрешить» → токен на экране. Совет для команды: заведите отдельный
   сервисный Яндекс-аккаунт, чтобы доступ не зависел от личного.

### Шаг 2. Развернуть

**Streamlit Community Cloud (бесплатно):** проект → приватный GitHub-репозиторий
(app.py в корне) → share.streamlit.io → New app → Secrets:

```
YANDEX_DISK_TOKEN = "ваш токен"
APP_PASSWORD = "пароль для входа"
```

**Или Docker на своем сервере:**

```
docker build -t dogovory .
docker run -d -p 8501:8501 -e YANDEX_DISK_TOKEN="…" -e APP_PASSWORD="…" dogovory
```

### Безопасность

- `APP_PASSWORD` обязателен: в форме персональные данные.
- Токен = полный доступ к Диску; в git не коммитить (`.gitignore` настроен).
